Floats

When you want an image and text to appear side-by-side, you want to `float` the image (usually) to the `left` or `right`.

The balloon to the right is float: right.

Once an element is floated, it is no longer part of the "document flow," which can often have unintended consequences.

Clearfix

A "clearfix" is a hack used to manage layouts with floats, see this example. The modern way to do this is by adding the CSS property overflow: auto; to the parent.

IDs

IDs work just like classes but are less flexible because an element can only have one ID. You also use a # instead of a . to select an element by ID.

Display: block & inline

Block elements are "100% wide" and do not let any other pieces of content share space on the same horizontal row.

Inline elements are as wide as their content and allow other elements to share space to the left and right.

div & span

When you can add whatever descripitors you want to an element via classes or IDs, the name of the element becomes less important. For example, you could have a paragraph that serves as the "introduction" to an article. If you had a <introduction> tag, you might use it instead. In cases like this, consider using a completely generic element like <span> and <div>.

File references

  • / at the beginning of a path means "go to the root". For example, /images/logo.png will look for a file in an images folder that is at the root. But root can mean different things. If you're serving a file up in Chrome with file:// then root means your hard drive. If you're running a web server, for example with python -m SimpleHTTPServer then root means your website project folder.
  • .. means "go up a folder"

It is recommended to references files in a "relative" way. So you have to ask yourself, "from the file I'm in, how do I get to the file I want to reference."
