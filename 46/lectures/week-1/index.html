Intro to HTML

  • What is HTML?
  • What is a markup language? As opposed to, say, a programming language?

The anatomy of an element

At its most essential, an element consists of an opening and a closing tag. Some tags don't close but we'll get to that in a minute.

  • Opening tag: Tells the browser that you're about to start describing some sort of content. It could be a paragraph, or a link, or anything that a browser understands.
  • Content: The stuff you're describing! Usually this ends up being the words on the page.
  • Closing tag: When you're done with the content, you just match the opening tag with a closing tag.

Practice: <p>, <strong>, <em>.

Nesting tags

Elements can be nested. And, in practice, that's a critical part of HTML. We refer to the whole page, or the entire HTML file as the document. And, as you can see, it forms a tree. There are rules about what elements can go inside other elements but it's not strongly enforced and mostly obvious.

Practice: <html>, <head>, <title>, <body>, <header>, <main>, <footer>, <h1>, <h2>, <h3>, <h4>, <h5>, <h6>, <ul>, <ol>, <li>.

Attributes

How would you go about linking to something? Even if you knew the element's tag was: <a>. Where would you put the link?

A complete list of all tags and their attributes can be found on the Mozilla Developer Network (MDN): HTML element reference.

Practice: <img>, <a>.

Intro to CSS

Anatomy of a selector

A CSS style rule consists of two parts:

  • Selector: Which elements do you want to target?
  • Declaration: What styles do you want those elements to have?

A declaration consists of two parts:

  • Property: What property are you trying to control on the element (e.g., color)?
  • Value: What's the value for that property (e.g., red)?

Commonly used properties

  • background
  • border
  • color
  • font
  • height
  • width
  • margin
  • padding
  • border-radius
  • box-shadow
  • text-align

Here is the complete CSS property reference: MDN CSS Property Reference.

Where does CSS go?

When writing CSS, you have three options of where to put it.

  1. Inline: With inline styles you use an attribute on the element you want to target. No selector is necessary.
    <p style="color: red;">I'm red text</p>
  2. Internal: Add CSS to the HTML page you're working on by opening a <style> tag and putting your CSS inside.
    <style>
    p { color: red; }
    </style>
    This code makes all paragraphs on that page red.
  3. External: Share styles between pages with an external stylesheet. Create a file in your project that ends with .css. Put your styles in that file. Then reference that file in your HTML with a link tag.
    <link rel="stylesheet" href="css/colors.css" media="screen" charset="utf-8">

What are the pros and cons of each method of including CSS?

Spruce up your homepage by adding some styles to it. Experiment with all three methods of including styles and try all the properties listed above.

Descendant CSS selectors

You can select nested elements by putting a space between selectors. For example: header a selects all the <a> tags in the header.

Some rules of thumb when nesting CSS selectors:

  • Keep it shallow! Deeply nested selectors can get confusing.
  • Omit any unnecessary selectors. Keep it as simple as possible!
