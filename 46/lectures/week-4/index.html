Intro to JavaScript

Why JavaScript?!

Before we dig into JavaScript (JS), let's just stop for a minute and ask ourselves, why do we need JS at all? We can already put content on a web page and style it. But as it stands our page doesn't do anything. And that's where JS comes into play. JS describes the behavior of your webpage.

You can add a button to the page with HTML. You can style it with CSS. But if you want something to happen when it's clicked, you'll need JS.

Object-Oriented Programming

Object-oriented programming (OOP) allows programmers to describe the world we live in with code. In OOP terms everything is an object. Whether it's a simple object like a number or something as complex as a flight plan, everything can be described as an object.

An object can be defined simply as a collection of properties and behavior. Properties are the nouns that serve to describe the object. Let's say we're describing a car as an object. A car has a make, model year, and color. These are properties of the car. But a car can also do things, like start, stop, and park. These are the functions of car and they describe how it behaves. Sometimes these behaviors are also called methods.

The template for an object is called a class. A class is what defines what properties and methods an object can have. It's like the recipe for all objects of that type. But a recipe doesn't actually give you anything to eat—it just describes the meal. In order to eat, you have to create the meal. The same is true in OOP. We call this process instantiating an object.

The document

On the web, the most useful object is the document. The document object describes the page. So it knows the page's title, the domain you're on, and it knows about all the elements on the page too.

Use the document object to:

  • Get the title of the page: document.title
  • Get the domain of the page: document.domain
  • Retrieve elements off the page: document.querySelector()
  • Create new elements: document.createElement()
Open Chrome's developer tools and switch to the Console tab now to try accessing the title and domain of your favorite website.

Functions

An object's methods are the functions it can perform. Functions almost always have verb names because they do stuff. And in order to do the work of a function, it has to be called, sometimes with and sometimes without parameters. Parameters are the input to a function. If you think of functions like a sausage maker, the meats would be the parameters. The output, or the return value of the function, would be a sausage.

Let's look at a real example. document has a method called querySelector() that will scan the page for an element matching a given CSS selector. That means this method takes one parameter, which you will pass into the parenthesis when you call it. It returns the first element it finds (or null if it doesn't find anything).

Variables

Up until now, you've seen the word var being used but we haven't talked about it yet. Variables, just like in math, allow us to refer to an eventual value. Variables are our own references to values that will be defined later. For example, if I'm writing a letter to my customers, I might say something like, "Dear <customer name>, Thank you for signing up...". In that letter, the "<customer name>" is just a stand in for the eventual value.

In JavaScript, we declare variables with the keyword var. A variable can be called whatever you'd like but it does have to follow certain rules.

Variable naming rules:

  • Variable names can't start with a number
  • Variable names cannot contain spaces
  • Variable names should camelCase
  • Variable names can not contain any special characters except "$" and "_"
  • Variable names can't be JavaScript keywords like var or new

Everything else is fair game!

You declare a variable in one of two ways.

Where does JavaScript go?

JavaScript is the only programming language your browser knows how execute without any additional plugins or extensions. It is what makes your browser so powerful. To tell your browser you'd like to execute some JavaScript code, simply open up a <script> tag and write some code.

For now, just remember put your JS after all the HTML on your page. This is because you'll be referencing HTML elements in your JavaScript. If you reference an element before it exists in the document you'll just get null back. How can you ask the browser for an element that isn't on the page yet?

You can also put your code in an external file and reference it that way.

Just like with an external CSS file, make sure there is no <script> tag inside your .js file.

Name Parameters Returns
prompt(message)

Asks someone a question by passing in a string.

For example:

Returns whatever the user typed in as a string. Even if they typed in a number 12, you'll get it back as 12.

Display a message to the user by passing in a string.

For example:

alert("Made you look.")

Note that the style of alerts and prompts is determined 100% by the browser. You have no control over them, which is why you don't see them much on the web.

Always returns undefined.

parseInt(string)

Convert the string representation of a number to an actual number.

For example:
parseInt("12")

Returns a number that you can do math with.

Remember, "12" + "23" equals "1234".

Numbers vs. Strings

In JavaScript "12" is not the same thing as 12. One is a string, a sequence of characters, and the other is a number. Without parseInt(), you'd never be able to do accurate addition. "Int" stands for "integer," which is a fancy word for whole number. There's also a function called parseFloat(), which will convert a string to a decimal number.

Creating New Elements

Imagine you want to put a comments box at the bottom of your web page. Unlike the calculator, you can no longer simply update a paragraph with the sum. Now, you have to create new elements and them to the page as people submit comments.

This is a two step process: creating the element then adding it to the page. And you need to learn two new functions to make it happen.

Name Parameters Returns
document.createElement(tagName)

Create a new element of a given tagName

For example:
document.createElement("p")
document.createElement("div")

Returns an element, just like document.querySelector(). The only different is that this element does not exist on the page yet.

parentEl.appendChild(childEl)

Adds an element to the page. Call this method on the parent element and pass in the child element. In this way you specify where you want the element to go.

Returns the child element as a convenience.

Once you've created a new element, it behaves just like every other element. So you can set the textContent or addEventListener() just like before.

Adding or Removing a Class from an Element

So you now know how to find an existing elements, create new elements, attach event listeners to them, and change their text. What about changing the CSS of an element to make it look a certain way?

For the same reason that inline styles are bad, setting CSS properties of an element with JS isn't recommended. But you can do it.

If you continue down this route, you'll find yourself writing a lot of CSS in your JS. Nobody on your team, including you, will ever think to look in the JS for styles. Instead, use clases. Declare your rules in CSS and then add/remove classes as needed using an element's classList object.

Setting an Attribute

What if you want to add an image tag to the page. You can, of course, use document.createElement("img") to create the element. But what about setting the src attribute? Elements like images or anchors are nothing without their attributes.

To set any attribute of any element, simply call the setAttribute() method. You can also remove an attribute altogether with removeAttribute().

Name Parameters Returns
el.setAttribute(attrName, attrValue)

Sets an attribute of the given element to the name and value you provide.

For example:

undefined

Frequently Asked Questions

How do I know when to use quotes?

In JS, "foo" and foo are two different things. The first is a string and the second is a variable. Strings are used whenever you want to refer to something by name or value. You'd use a string to put text on the page (el.textContent = "hello";), reference a class name (document.querySelector(".products")), or specify a type of event ("click"). When you use a string you know its value. But when you reference something without quotes, it's value is dynamic. That's why we call it a variable, because it's value changes based on what a user typed in or what results came back from a database. If you don't put quotes around something, JS will assume that you've defined that thing elsewhere either with var or function. So a good rule of thumb is if it isn't in quotes there better be a var or function that defines it somewhere.
